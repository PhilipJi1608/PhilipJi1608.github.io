- An analogy to describe the differences between HTML and CSS

Picture a live animal, like a zebra. HTML is the bone structure of the zebra, the muscles, tendons, and did I mention, the backbone of the zebra? CSS is the stripes. Stylish af! I know. Fancy lad, that one. Now, obviously zebra embodies the perfect blend of beauty and the beast, which is quite alike a webpage. Without HTML, there is no text, no structure, nothing to go on. - Describe what the DOM is and an example of how you might interact with it.

DOM is how a web browser sees the code for a web page. It can be interacted (tinkered) with using DevTools in Chrome, or pressing F12. Many many things can be tinkered with to achieve maximum hilarity. Remember the zebra? It's the brain of the zebra. Sure, without it, the body can move on its own out of pure electrical impulses, but with it, the zebra becomes much, much more powerful and dynamic.

- Explain the difference between accessing data from arrays and objects.

Arrays are basically a list of data stored together. Objects are much more sophisticated, often with functions and many different types of data stored within. You can access a single piece of information or a dozen from an object. Arrays are much simpler.

- Explain what functions are and why they are useful.

Functions are set of commands created to enable us to recall and use at a later time. Let's say if I made an object that records the total number of zebras in an area, I can use a function to pull up that specific information. If I wanted to find out the number of zebras that survive from predators each year, I can also create a function that tells me about the number of surviving zebras, how many attacks occured, and how many daring escapes each zebra has made.
